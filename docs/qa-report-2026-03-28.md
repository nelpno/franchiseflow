## BUGS CORRIGIDOS

### [BUG-001] Dashboard Admin stats nao filtravam por periodo — CORRIGIDO
- **Antes**: Stats cards mostravam mesmos numeros (5 vendas, R$ 458) em 7d e 30d
- **Fix**: Substituiu DailySummary (vazio) por queries diretas Sale.list/Contact.list com limit 2000
- **Depois**: 7d mostra 101 vendas, R$ 8.687 — bate com Reports
- **Arquivo**: `src/components/dashboard/AdminDashboard.jsx`

### [BUG-003] Estoque negativo (-1, -2) em 4 produtos — CORRIGIDO
- **Antes**: 4 itens com estoque negativo em 3 franquias
- **Fix**: Corrigido diretamente via Supabase REST API (PATCH quantity=0)
- **Produtos afetados**: Massa de Pastel 1kg (Araraquara), Rondelli (Sao Carlos), Sofioli (Bauru, Sao Paulo)

### [BUG-004] Acentos faltando na pagina Gestao — CORRIGIDO
- **Fix**: "Evolucao" → "Evolucao", "graficos" → "graficos", "Sugestao" → "Sugestao"
- **Arquivos**: `ResultadoCharts.jsx`, `TabEstoque.jsx`, `PurchaseOrderForm.jsx`

## MELHORIAS UX IMPLEMENTADAS

### [UX-001] OnboardingWelcome — conteudo centralizado verticalmente
- **Antes**: Conteudo no topo, botao "Proximo" isolado no rodape
- **Fix**: `items-start` → `items-center` no flex container
- **Arquivo**: `src/pages/OnboardingWelcome.jsx`

### [UX-002] Label "Valor liquido" renomeado
- **Antes**: "Valor liquido" (terminologia proibida pelo CLAUDE.md)
- **Depois**: "Total a receber"
- **Arquivo**: `src/components/minha-loja/SaleForm.jsx`

### [UX-003] Produtos sem estoque separados no SaleForm
- **Antes**: 28 produtos misturados, sem estoque no meio dos com estoque
- **Depois**: Produtos com estoque primeiro, sem estoque depois (visualmente desbotados opacity-40)
- **Arquivo**: `src/components/minha-loja/SaleForm.jsx` (ProductSearch)

### [UX-005] Health Score thresholds + Acompanhamento counter
- **Antes**: Thresholds 70/40 (errado), counter somava "nova" como "saudavel" (40 saudaveis falso)
- **Depois**: Thresholds 80/50 (correto), counter separa "Novas" com icone fiber_new em azul
- **Arquivos**: `src/lib/healthScore.js`, `src/pages/Acompanhamento.jsx`

### [UX-006] Periodo "Nova" reduzido de 30 para 14 dias
- **Antes**: Franquias marcadas "nova" por 30 dias (escondia problemas reais)
- **Depois**: 14 dias — tempo suficiente para onboarding, depois classifica normalmente
- **Arquivo**: `src/lib/healthScore.js`

### [UX-007] Ranking Dashboard — franquias R$0 ocultadas
- **Antes**: Top 5 mostrava 4 franquias com R$ 0,00 (poluicao visual)
- **Depois**: Mostra apenas franquias com vendas + nota discreta "X franquias sem vendas no periodo"
- **Arquivo**: `src/components/dashboard/FranchiseRanking.jsx`
